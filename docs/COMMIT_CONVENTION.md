# Commit Message Convention

Format: `<type>: <description>`

- **feat**: Adds a new feature
- **fix**: Fixes a bug
- **refactor**: Rewrites/restructures code (no new features or bug fixes)
- **chore**: Miscellaneous changes (dependencies, .gitignore, etc.)
- **perf**: Performance improvements
- **ci**: Continuous integration related
- **ops**: Infrastructure, deployment, or backups
- **build**: Build system, tools, or project versioning
- **docs**: Documentation updates (e.g., README)
- **style**: Code formatting (whitespace, semicolons, etc.)
- **revert**: Reverts a previous commit
- **test**: Adds or corrects tests

---
### Examples:
- feat: add QR scanner component
- docs: update setup instructions
- chore: install postgres driver
